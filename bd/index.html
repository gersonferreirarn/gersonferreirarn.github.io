Módulo 1 – Fundamentos de SQL

40 minutos Básico 4 lições

Conceito de Linguagem de Consulta Estruturada (SQL)

SQL (Structured Query Language) é uma linguagem padrão para gerenciar e manipular bancos de dados relacionais. Desenvolvida na década de 1970 pela IBM, tornou-se o padrão ISO para sistemas de gerenciamento de banco de dados.

SQL permite que você:

  • Consultar dados de um banco de dados
  • Inserir, atualizar e excluir registros
  • Criar e modificar estruturas de banco de dados
  • Controlar o acesso aos dados

Ponto-chave

SQL é declarativa, não procedural. Você especifica o que deseja, não como obtê-lo.

Categorias de comandos SQL

Os comandos SQL são divididos em várias categorias funcionais:

  • DQL (Data Query Language) - SELECT: para consultar dados
  • DML (Data Manipulation Language) - INSERT, UPDATE, DELETE: para manipular dados
  • DDL (Data Definition Language) - CREATE, ALTER, DROP: para definir estruturas
  • DCL (Data Control Language) - GRANT, REVOKE: para controlar acessos
  • TCL (Transaction Control Language) - COMMIT, ROLLBACK: para gerenciar transações
-- Exemplo de DQL SELECT nome, email FROM clientes WHERE status = 'ativo'; -- Exemplo de DML UPDATE produtos SET preco = preco * 1.1 WHERE categoria = 'eletrônicos';

Estrutura básica de instruções

A maioria das instruções SQL segue uma estrutura padrão:

SELECT coluna1, coluna2 FROM tabela WHERE condição GROUP BY coluna HAVING condição_grupo ORDER BY coluna LIMIT número;

Componentes principais:

  • SELECT - Especifica as colunas a serem retornadas
  • FROM - Especifica a tabela de origem
  • WHERE - Filtra registros baseados em condições
  • GROUP BY - Agrupa registros por colunas
  • ORDER BY - Ordena os resultados

Primeiros passos com SQL

Para começar a usar SQL, você precisa:

  • Instalar um sistema de gerenciamento de banco de dados (MySQL, PostgreSQL, SQL Server, etc.)
  • Conectar-se ao banco de dados usando uma ferramenta cliente
  • Criar um banco de dados e tabelas
  • Inserir dados e começar a fazer consultas
-- Criando um banco de dados CREATE DATABASE meu_banco; -- Usando o banco de dados USE meu_banco; -- Criando uma tabela simples CREATE TABLE usuarios ( id INT PRIMARY KEY AUTO_INCREMENT, nome VARCHAR(100) NOT NULL, email VARCHAR(100) UNIQUE );

Módulo 2 – Definição de Dados (DDL)

45 minutos Intermediário 4 lições

Introdução ao DDL

A Linguagem de Definição de Dados (DDL) é um subconjunto do SQL usado para definir e modificar estruturas de banco de dados. Comandos DDL permitem criar, alterar e excluir objetos como tabelas, índices e esquemas.

Os principais comandos DDL são:

  • CREATE - Para criar novos objetos
  • ALTER - Para modificar objetos existentes
  • DROP - Para remover objetos
  • TRUNCATE - Para remover todos os dados de uma tabela

Ponto-chave

Comandos DDL são autoconfirmáveis - não é possível desfazer suas operações através de ROLLBACK na maioria dos sistemas de banco de dados.

CREATE: criação de estruturas

O comando CREATE é usado para criar novos objetos no banco de dados. O objeto mais comum criado com DDL é a tabela.

CREATE TABLE clientes ( id INT PRIMARY KEY, nome VARCHAR(100) NOT NULL, email VARCHAR(100) UNIQUE, data_cadastro DATE DEFAULT CURRENT_DATE );

Outros objetos que podem ser criados com CREATE:

  • CREATE INDEX - Para criar índices
  • CREATE VIEW - Para criar visualizações
  • CREATE PROCEDURE - Para criar procedimentos armazenados
  • CREATE TRIGGER - Para criar gatilhos

ALTER: modificação de estruturas

O comando ALTER permite modificar a estrutura de objetos existentes. Com ele, você pode adicionar, modificar ou excluir colunas de uma tabela.

-- Adicionar uma nova coluna ALTER TABLE clientes ADD COLUMN telefone VARCHAR(20); -- Modificar tipo de dados de uma coluna ALTER TABLE clientes MODIFY COLUMN nome VARCHAR(150); -- Excluir uma coluna ALTER TABLE clientes DROP COLUMN telefone;

Atenção!

Alterações em tabelas com grandes volumes de dados podem causar bloqueios e lentidão. Planeje cuidadosamente essas operações.

DROP: remoção de estruturas

O comando DROP remove permanentemente objetos do banco de dados. Esta operação é irreversível na maioria dos sistemas.

-- Remover uma tabela DROP TABLE clientes; -- Remover um índice DROP INDEX idx_clientes_nome; -- Remover uma view DROP VIEW vw_clientes_ativos;

Para remover apenas os dados de uma tabela mantendo sua estrutura, use TRUNCATE:

TRUNCATE TABLE log_acessos;

Cuidado!

O comando DROP remove permanentemente o objeto e todos os seus dados. Use com extrema cautela.

Módulo 3 – Criação e Manipulação de Tabelas

50 minutos Intermediário 4 lições

Sintaxe do CREATE TABLE

O comando CREATE TABLE é usado para criar novas tabelas no banco de dados. A sintaxe básica inclui o nome da tabela e a definição de suas colunas.

CREATE TABLE nome_tabela ( coluna1 tipo_dado [restrições], coluna2 tipo_dado [restrições], ... [restrições_de_tabela] );

Exemplo completo:

CREATE TABLE funcionarios ( id INT PRIMARY KEY AUTO_INCREMENT, nome VARCHAR(100) NOT NULL, cpf VARCHAR(14) UNIQUE NOT NULL, data_nascimento DATE, salario DECIMAL(10,2) CHECK (salario > 0), departamento_id INT, data_contratacao DATE DEFAULT CURRENT_DATE, FOREIGN KEY (departamento_id) REFERENCES departamentos(id) );

Definição de colunas

Cada coluna em uma tabela deve ter um nome e um tipo de dados. Além disso, podem ser aplicadas restrições e valores padrão.

Elementos na definição de colunas:

  • Nome da coluna - Identificador único na tabela
  • Tipo de dados - Define o tipo de informação armazenada
  • Restrições - Regras que os dados devem seguir
  • Valor padrão - Valor usado quando nenhum é especificado
CREATE TABLE produtos ( id INT PRIMARY KEY AUTO_INCREMENT, nome VARCHAR(100) NOT NULL, descricao TEXT, preco DECIMAL(10,2) NOT NULL, estoque INT DEFAULT 0, ativo BOOLEAN DEFAULT TRUE );

Tipos de dados mais comuns

Os tipos de dados definem que tipo de informação pode ser armazenada em cada coluna:

  • Numéricos: INT, DECIMAL, FLOAT
  • Texto: VARCHAR, CHAR, TEXT
  • Data/Hora: DATE, TIME, DATETIME
  • Booleanos: BOOLEAN, BIT
  • Binários: BLOB, BINARY
CREATE TABLE exemplos ( id INT, nome VARCHAR(50), descricao TEXT, preco DECIMAL(10,2), data_criacao DATE, ativo BOOLEAN, imagem BLOB );

Boas práticas de modelagem física

Ao criar tabelas, siga estas boas práticas:

  • Use nomes descritivos - Seja claro e consistente
  • Defina chaves primárias - Cada tabela deve ter uma
  • Normalize dados - Evite redundância
  • Use restrições apropriadas - Garanta integridade dos dados
  • Documente suas tabelas - Use comentários quando necessário

Dica profissional

Use ferramentas de modelagem como ERD (Entity-Relationship Diagram) para planejar suas tabelas antes de implementá-las.

Módulo 4 – Tipos de Dados

40 minutos Intermediário 5 lições

Tipos numéricos

Tipos numéricos são usados para armazenar valores numéricos, sejam inteiros ou decimais:

  • INT - Números inteiros (-2.147.483.648 a 2.147.483.647)
  • BIGINT - Inteiros grandes (-9.223.372.036.854.775.808 a 9.223.372.036.854.775.807)
  • DECIMAL(p,s) - Números decimais exatos (p = precisão, s = escala)
  • FLOAT - Números de ponto flutuante de precisão simples
  • DOUBLE - Números de ponto flutuante de precisão dupla
CREATE TABLE produtos ( id INT PRIMARY KEY, nome VARCHAR(100), preco DECIMAL(10,2), -- 10 dígitos totais, 2 decimais peso FLOAT, estoque INT DEFAULT 0 );

Tipos de texto

Tipos de texto armazenam caracteres e strings:

  • CHAR(n) - String de comprimento fixo (até 255 caracteres)
  • VARCHAR(n) - String de comprimento variável (até 65.535 caracteres)
  • TEXT - Texto longo (até 65.535 caracteres)
  • MEDIUMTEXT - Texto muito longo (até 16.777.215 caracteres)
  • LONGTEXT - Texto extremamente longo (até 4.294.967.295 caracteres)
CREATE TABLE artigos ( id INT PRIMARY KEY, titulo VARCHAR(200), resumo VARCHAR(500), conteudo TEXT, autor VARCHAR(100) );

Tipos de data e hora

Esses tipos armazenam informações temporais:

  • DATE - Data no formato YYYY-MM-DD
  • TIME - Hora no formato HH:MM:SS
  • DATETIME - Data e hora combinadas
  • TIMESTAMP - Data e hora com fuso horário
  • YEAR - Ano no formato YYYY
CREATE TABLE eventos ( id INT PRIMARY KEY, nome VARCHAR(100), data_evento DATE, hora_inicio TIME, data_criacao DATETIME DEFAULT CURRENT_TIMESTAMP );

Tipos especiais

Alguns tipos de dados especiais incluem:

  • BOOLEAN/BOOL - Valores lógicos (TRUE/FALSE)
  • ENUM - Enumeração de valores pré-definidos
  • SET - Conjunto de valores pré-definidos
  • BLOB - Dados binários grandes
  • JSON - Dados no formato JSON
CREATE TABLE usuarios ( id INT PRIMARY KEY, nome VARCHAR(100), ativo BOOLEAN DEFAULT TRUE, tipo ENUM('admin', 'editor', 'visitante'), preferencias JSON, foto BLOB );

Escolha adequada de tipos de dados

A escolha correta dos tipos de dados é crucial para:

  • Otimizar espaço - Use o menor tipo que atenda às necessidades
  • Garantir integridade - Tipos adequados previnem erros
  • Melhorar performance - Tipos corretos otimizam consultas

Recomendações

Use INT para IDs, VARCHAR para textos variáveis, DECIMAL para valores monetários e DATETIME para registros temporais.

Módulo 5 – Restrições (Constraints)

45 minutos Intermediário 6 lições

Finalidade das restrições

Restrições são regras aplicadas a colunas ou tabelas para garantir a integridade dos dados. Elas asseguram que os dados inseridos atendam a critérios específicos.

Principais benefícios das restrições:

  • Garantem consistência dos dados
  • Previnem inserção de dados inválidos
  • Definem relacionamentos entre tabelas
  • Impõem regras de negócio no nível do banco

Ponto-chave

Restrições são implementadas no nível do banco de dados, garantindo integridade independentemente da aplicação.

PRIMARY KEY

A chave primária identifica exclusivamente cada registro em uma tabela:

  • Deve ser única para cada registro
  • Não pode conter valores NULL
  • Uma tabela só pode ter uma chave primária
-- Definindo chave primária na criação CREATE TABLE clientes ( id INT PRIMARY KEY, nome VARCHAR(100) ); -- Ou após a criação ALTER TABLE clientes ADD PRIMARY KEY (id);

FOREIGN KEY

A chave estrangeira estabelece um relacionamento entre duas tabelas:

  • Referencia uma chave primária em outra tabela
  • Garante integridade referencial
  • Pode ter ações ON DELETE e ON UPDATE
CREATE TABLE pedidos ( id INT PRIMARY KEY, cliente_id INT, data_pedido DATE, FOREIGN KEY (cliente_id) REFERENCES clientes(id) ON DELETE CASCADE ON UPDATE CASCADE );

UNIQUE

A restrição UNIQUE garante que todos os valores em uma coluna sejam diferentes:

  • Permite valores NULL (diferente de PRIMARY KEY)
  • Pode ser aplicada a múltiplas colunas
  • Uma tabela pode ter múltiplas restrições UNIQUE
CREATE TABLE usuarios ( id INT PRIMARY KEY, email VARCHAR(100) UNIQUE, cpf VARCHAR(14) UNIQUE );

NOT NULL e DEFAULT

NOT NULL garante que uma coluna não possa conter valores NULL:

CREATE TABLE produtos ( id INT PRIMARY KEY, nome VARCHAR(100) NOT NULL, descricao TEXT );

DEFAULT define um valor padrão quando nenhum é especificado:

CREATE TABLE pedidos ( id INT PRIMARY KEY, status VARCHAR(20) DEFAULT 'pendente', data_criacao DATETIME DEFAULT CURRENT_TIMESTAMP );

CHECK

A restrição CHECK garante que os valores em uma coluna atendam a uma condição específica:

CREATE TABLE funcionarios ( id INT PRIMARY KEY, nome VARCHAR(100), salario DECIMAL(10,2) CHECK (salario > 0), idade INT CHECK (idade >= 18) );

Condições complexas também são possíveis:

CREATE TABLE produtos ( id INT PRIMARY KEY, nome VARCHAR(100), preco DECIMAL(10,2), estoque INT, CHECK (preco > 0 AND estoque >= 0) );

Módulo 6 – Índices

40 minutos Avançado 5 lições

Conceito de índices

Índices são estruturas de dados que melhoram a velocidade de operações de recuperação de dados em uma tabela. Funcionam como o índice de um livro, permitindo encontrar informações rapidamente.

Características principais:

  • Estruturas de dados especiais
  • Armazenam ponteiros para os dados reais
  • São mantidos automaticamente pelo banco de dados
  • Ocupam espaço adicional em disco

Como funcionam

Índices usam estruturas como B-trees para organizar dados de forma ordenada, permitindo buscas binárias eficientes.

Benefícios e impacto no desempenho

Benefícios dos índices:

  • Aceleram consultas SELECT
  • Melhoram performance de JOINs
  • Otimizam cláusulas WHERE e ORDER BY
  • Agilizam buscas por valores específicos

Impactos negativos:

  • Consomem espaço em disco
  • Desaceleram operações de INSERT, UPDATE e DELETE
  • Requerem manutenção adicional

Trade-off

Índices melhoram a leitura mas pioram a escrita. É preciso encontrar o equilíbrio correto.

Criação de índices (CREATE INDEX)

Para criar um índice, use o comando CREATE INDEX:

-- Criar índice simples CREATE INDEX idx_clientes_nome ON clientes(nome); -- Criar índice composto CREATE INDEX idx_pedidos_cliente_data ON pedidos(cliente_id, data_pedido); -- Criar índice único CREATE UNIQUE INDEX idx_emails ON usuarios(email);

Você também pode criar índices durante a criação da tabela:

CREATE TABLE produtos ( id INT PRIMARY KEY, nome VARCHAR(100), categoria VARCHAR(50), INDEX idx_produtos_categoria (categoria) );

Índices únicos

Índices únicos garantem que não haja valores duplicados nas colunas indexadas:

  • Impõem restrição de unicidade
  • São automaticamente criados para PRIMARY KEY e UNIQUE
  • Podem ser criados manualmente
-- Criando índice único CREATE UNIQUE INDEX idx_cpf ON funcionarios(cpf); -- Tentativa de inserir duplicata falhará INSERT INTO funcionarios (cpf) VALUES ('123.456.789-00'); INSERT INTO funcionarios (cpf) VALUES ('123.456.789-00'); -- Erro!

Quando usar e quando evitar índices

Use índices para:

  • Colunas frequentemente usadas em WHERE
  • Colunas em cláusulas ORDER BY
  • Colunas em JOINs
  • Colunas com alta seletividade (muitos valores distintos)

Evite índices para:

  • Tabelas pequenas
  • Colunas raramente consultadas
  • Colunas com muitos valores NULL
  • Tabelas com muitas operações de escrita

Dica profissional

Use EXPLAIN para analisar o plano de execução das consultas e identificar onde os índices estão sendo utilizados.
