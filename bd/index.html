1.1 Banco de dados, SGBD e sistema de banco de dados

Banco de Dados (BD)

Uma coleção organizada de dados inter-relacionados, representando informações sobre um domínio específico.

Exemplo: Lista de clientes de uma empresa

SGBD

Sistema de Gerenciamento de Banco de Dados - Software que permite criar, manipular e administrar o BD.

Exemplos: MySQL, PostgreSQL, Oracle

Sistema de BD

Conjunto completo incluindo BD, SGBD, usuários, aplicações e procedimentos.

Componentes: Dados + Software + Hardware + Usuários

Relação entre os Componentes

Aplicações / Usuários

Banco de Dados (Dados)

1.2 Vantagens de usar um SGBD

Controle de Redundância

Evita duplicação desnecessária de dados, economizando espaço e garantindo consistência.

Exemplo: Dados do cliente armazenados uma única vez

Segurança

Controle de acesso granular com diferentes níveis de permissão para usuários.

Exemplo: Apenas RH pode ver salários

Integridade

Garante que os dados sigam regras de negócio e restrições definidas.

Exemplo: Impedir idade negativa

Recuperação

Mecanismos de backup e recuperação automáticos em caso de falhas.

Exemplo: Restaurar após queda de energia

1.3 Elementos Fundamentais

Campos, Registros e Tabelas

Campos (Colunas)

Unidades básicas de informação que representam características específicas.

Registros (Linhas)

Conjunto de valores que representam uma instância completa de dados.

Tabelas

Estrutura que organiza dados em linhas e colunas.

ID Nome Email Idade
1 João Silva joao@email.com 28
2 Maria Santos maria@email.com 32

Índices

Estruturas que melhoram a velocidade de operações de recuperação de dados, funcionando como o índice de um livro.

Benefícios
  • • Buscas mais rápidas
  • • Ordenação eficiente
  • • Melhora performance de JOINs
Desvantagens
  • • Consome espaço extra
  • • Lentidão em inserções
  • • Requer manutenção

Tipos de Chaves

Chave Primária (PK)

Identifica exclusivamente cada registro. Única e não nula.

Chave Estrangeira (FK)

Referencia PK de outra tabela. Estabelece relacionamentos.

Chave Candidata

Qualquer coluna que poderia servir como PK.

Tipos de Integridade

Integridade de Entidade

Garante que cada tabela tenha uma PK única e não nula. Nenhum registro duplicado.

Integridade Referencial

Assegura que FKs sempre referenciem PKs válidas existentes.

Integridade de Domínio

Especifica que valores em uma coluna pertençam a um domínio definido (tipo, formato, intervalo).
