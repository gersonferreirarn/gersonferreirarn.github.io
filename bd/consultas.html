SELECT e FROM

30 minutos Básico 3 lições

Comando SELECT

O comando SELECT é usado para recuperar dados de um banco de dados. É o comando mais fundamental em SQL para consulta de informações.

A sintaxe básica do SELECT é:

SELECT coluna1, coluna2, ... FROM tabela;

Para selecionar todas as colunas de uma tabela, use o asterisco (*) como atalho:

SELECT * FROM tabela;

Ponto-chave

O SELECT sempre retorna um conjunto de resultados (result set) com os dados solicitados.

Comando FROM

O comando FROM especifica a tabela ou tabelas de onde os dados serão recuperados. Ele sempre acompanha o comando SELECT.

Exemplos de uso:

-- Selecionar colunas específicas SELECT nome, email, telefone FROM clientes; -- Selecionar todas as colunas SELECT * FROM produtos;

O FROM também pode ser usado com aliases (apelidos) para simplificar a escrita de consultas complexas:

SELECT c.nome, p.descricao FROM clientes AS c, pedidos AS p WHERE c.id = p.cliente_id;

Combinando SELECT e FROM

A combinação de SELECT e FROM forma a base de qualquer consulta SQL. Vamos ver alguns exemplos práticos:

-- Exemplo 1: Selecionar informações de clientes SELECT id, nome, email, data_cadastro FROM clientes; -- Exemplo 2: Selecionar produtos com preço acima de 100 SELECT nome, preco, estoque FROM produtos; -- Exemplo 3: Selecionar pedidos com informações do cliente SELECT p.id, p.data_pedido, c.nome, c.email FROM pedidos AS p JOIN clientes AS c ON p.cliente_id = c.id;

Dica profissional

Evite usar SELECT * em código de produção. Especifique apenas as colunas necessárias para melhorar a performance e a clareza do código.

WHERE

35 minutos Básico 3 lições

Introdução ao WHERE

O comando WHERE é usado para filtrar registros que atendem a condições específicas. Ele permite restringir os resultados retornados por uma consulta.

A sintaxe básica é:

SELECT coluna1, coluna2, ... FROM tabela WHERE condição;

A condição no WHERE pode usar diversos operadores que veremos nos próximos módulos.

Ponto-chave

O WHERE é aplicado antes do agrupamento e ordenação dos dados, filtrando os registros individualmente.

Exemplos básicos com WHERE

Vamos ver alguns exemplos práticos do uso do WHERE:

-- Filtrar clientes por ID SELECT * FROM clientes WHERE id = 10; -- Filtrar produtos por categoria SELECT nome, preco FROM produtos WHERE categoria = 'Eletrônicos'; -- Filtrar pedidos por data SELECT id, valor FROM pedidos WHERE data_pedido = '2023-05-15';

O WHERE também pode ser combinado com outros comandos SQL:

-- Atualizar registros específicos UPDATE produtos SET preco = preco * 1.1 WHERE categoria = 'Eletrônicos'; -- Excluir registros específicos DELETE FROM logs WHERE data_criacao < '2022-01-01';

Múltiplas condições com WHERE

O WHERE permite combinar múltiplas condições usando operadores lógicos como AND e OR:

-- Usando AND (todas as condições devem ser verdadeiras) SELECT nome, email FROM clientes WHERE cidade = 'São Paulo' AND status = 'ativo'; -- Usando OR (pelo menos uma condição deve ser verdadeira) SELECT nome, email FROM clientes WHERE cidade = 'São Paulo' OR cidade = 'Rio de Janeiro'; -- Combinando AND e OR com parênteses SELECT nome, email FROM clientes WHERE (cidade = 'São Paulo' OR cidade = 'Rio de Janeiro') AND status = 'ativo';

Atenção!

Quando combinar AND e OR, use parênteses para garantir a ordem correta de avaliação das condições.

Operadores Relacionais

30 minutos Básico 3 lições

Introdução aos Operadores Relacionais

Operadores relacionais são usados para comparar valores em condições do WHERE. Eles retornam verdadeiro ou falso, determinando quais registros serão incluídos no resultado.

Os principais operadores relacionais em SQL são:

  • = - Igual a
  • != ou <> - Diferente de
  • > - Maior que
  • < - Menor que
  • >= - Maior ou igual a
  • <= - Menor ou igual a

Ponto-chave

Operadores relacionais são a base para a maioria das condições em cláusulas WHERE.

Exemplos com Operadores Relacionais

Vamos ver exemplos práticos com cada operador:

-- Igual a (=) SELECT * FROM produtos WHERE preco = 99.90; -- Diferente de (!= ou <>) SELECT * FROM produtos WHERE categoria != 'Eletrônicos'; -- Maior que (>) SELECT * FROM produtos WHERE preco > 100; -- Menor que (<) SELECT * FROM produtos WHERE estoque < 10; -- Maior ou igual a (>=) SELECT * FROM funcionarios WHERE salario >= 3000; -- Menor ou igual a (<=) SELECT * FROM pedidos WHERE valor <= 50;

Combinando Operadores Relacionais

Operadores relacionais podem ser combinados com operadores lógicos para criar condições mais complexas:

-- Faixa de preço entre 50 e 100 SELECT nome, preco FROM produtos WHERE preco >= 50 AND preco <= 100; -- Produtos fora da faixa de preço SELECT nome, preco FROM produtos WHERE preco < 50 OR preco > 100; -- Clientes de cidades específicas com idade mínima SELECT nome, cidade, idade FROM clientes WHERE (cidade = 'São Paulo' OR cidade = 'Rio de Janeiro') AND idade >= 18;

Dica profissional

Para verificar se um valor está dentro de uma faixa, você pode usar o operador BETWEEN, que veremos em um módulo posterior.

Operadores Lógicos

35 minutos Intermediário 4 lições

Introdução aos Operadores Lógicos

Operadores lógicos são usados para combinar múltiplas condições em uma cláusula WHERE. Eles permitem criar consultas mais complexas e específicas.

Os principais operadores lógicos em SQL são:

  • AND - Retorna verdadeiro se todas as condições forem verdadeiras
  • OR - Retorna verdadeiro se pelo menos uma condição for verdadeira
  • NOT - Inverte o resultado da condição

Ponto-chave

Operadores lógicos seguem a precedência: NOT é avaliado primeiro, depois AND, e finalmente OR. Use parênteses para alterar essa ordem.

Operador AND

O operador AND exige que todas as condições sejam verdadeiras para que o registro seja incluído no resultado:

-- Clientes ativos de São Paulo SELECT * FROM clientes WHERE cidade = 'São Paulo' AND status = 'ativo'; -- Produtos com preço entre 50 e 100 e estoque disponível SELECT * FROM produtos WHERE preco >= 50 AND preco <= 100 AND estoque > 0; -- Pedidos recentes de alto valor SELECT * FROM pedidos WHERE data_pedido >= '2023-01-01' AND valor > 1000;

Operador OR

O operador OR retorna registros que atendem a pelo menos uma das condições:

-- Clientes de São Paulo ou Rio de Janeiro SELECT * FROM clientes WHERE cidade = 'São Paulo' OR cidade = 'Rio de Janeiro'; -- Produtos das categorias Eletrônicos ou Informática SELECT * FROM produtos WHERE categoria = 'Eletrônicos' OR categoria = 'Informática'; -- Pedidos com status específicos SELECT * FROM pedidos WHERE status = 'pendente' OR status = 'processando';

Operador NOT e Combinações

O operador NOT inverte o resultado de uma condição. Ele também pode ser combinado com outros operadores lógicos:

-- Clientes que não são de São Paulo SELECT * FROM clientes WHERE NOT cidade = 'São Paulo'; -- Produtos que não estão na categoria Eletrônicos SELECT * FROM produtos WHERE NOT categoria = 'Eletrônicos'; -- Combinação de AND, OR e NOT SELECT * FROM clientes WHERE (cidade = 'São Paulo' OR cidade = 'Rio de Janeiro') AND NOT status = 'inativo';

Atenção!

Quando combinar AND e OR, use parênteses para garantir a ordem correta de avaliação. Sem parênteses, AND tem precedência sobre OR.

LIKE

30 minutos Intermediário 3 lições

Introdução ao LIKE

O operador LIKE é usado em cláusulas WHERE para buscar padrões em colunas de texto. Ele é mais flexível que o operador de igualdade (=), permitindo buscas parciais.

O LIKE utiliza dois caracteres curinga principais:

  • % - Representa zero, um ou múltiplos caracteres
  • _ - Representa exatamente um caractere

Ponto-chave

O LIKE é sensível a maiúsculas e minúsculas em alguns bancos de dados, mas não em outros. Verifique a documentação do seu SGBD.

Usando o curinga %

O curinga % é o mais utilizado com o LIKE. Veja exemplos:

-- Começa com 'Silva' SELECT * FROM clientes WHERE nome LIKE 'Silva%'; -- Termina com 'silva' SELECT * FROM clientes WHERE nome LIKE '%silva'; -- Contém 'silva' em qualquer posição SELECT * FROM clientes WHERE nome LIKE '%silva%'; -- Produtos cuja descrição contém 'HD' SELECT * FROM produtos WHERE descricao LIKE '%HD%';

Dica profissional

Buscas que começam com curinga (LIKE '%texto') são mais lentas que buscas que terminam com curinga (LIKE 'texto%'), pois não podem usar índices eficientemente.

Usando o curinga _ e combinações

O curinga _ representa exatamente um caractere. Ele pode ser combinado com % para criar padrões mais específicos:

-- Nomes com 5 letras que começam com 'Maria' SELECT * FROM clientes WHERE nome LIKE 'Maria_'; -- CEPs no formato 99999-999 SELECT * FROM clientes WHERE cep LIKE '_____-___'; -- Combinação de curingas -- Produtos cujo código começa com 'A', tem qualquer caractere -- em seguida e termina com '5' SELECT * FROM produtos WHERE codigo LIKE 'A_%5';

Para buscar os próprios caracteres curinga, use a cláusula ESCAPE:

-- Buscar nomes que contêm exatamente 20% SELECT * FROM produtos WHERE nome LIKE '%20\%%' ESCAPE '\';

BETWEEN e IN

30 minutos Intermediário 4 lições

Operador BETWEEN

O operador BETWEEN é usado para selecionar valores dentro de um intervalo. Os valores podem ser números, texto ou datas.

A sintaxe do BETWEEN é:

SELECT coluna(s) FROM tabela WHERE coluna BETWEEN valor1 AND valor2;

É importante notar que o BETWEEN é inclusivo, ou seja, inclui os valores de início e fim do intervalo.

Ponto-chave

O BETWEEN é equivalente a usar "coluna >= valor1 AND coluna <= valor2", mas torna a consulta mais legível.

Exemplos com BETWEEN

Vamos ver exemplos práticos do uso do BETWEEN com diferentes tipos de dados:

-- Intervalo de preços SELECT nome, preco FROM produtos WHERE preco BETWEEN 50 AND 100; -- Intervalo de datas SELECT nome, data_pedido FROM pedidos WHERE data_pedido BETWEEN '2023-01-01' AND '2023-01-31'; -- Intervalo alfabético SELECT nome, sobrenome FROM clientes WHERE sobrenome BETWEEN 'A' AND 'M';

O BETWEEN também pode ser combinado com NOT para selecionar valores fora do intervalo:

-- Produtos com preço fora da faixa SELECT nome, preco FROM produtos WHERE preco NOT BETWEEN 50 AND 100;

Operador IN

O operador IN permite especificar múltiplos valores em uma cláusula WHERE. É uma alternativa mais limpa ao uso de múltiplos OR.

A sintaxe do IN é:

SELECT coluna(s) FROM tabela WHERE coluna IN (valor1, valor2, ...);

Exemplos de uso do IN:

-- Clientes de cidades específicas SELECT * FROM clientes WHERE cidade IN ('São Paulo', 'Rio de Janeiro', 'Belo Horizonte'); -- Produtos de categorias específicas SELECT * FROM produtos WHERE categoria IN ('Eletrônicos', 'Informática', 'Celulares'); -- Pedidos com status específicos SELECT * FROM pedidos WHERE status IN ('pendente', 'processando', 'enviado');

IN com subconsultas e NOT IN

O operador IN pode ser usado com subconsultas, permitindo comparar valores com o resultado de outra consulta:

-- Clientes que fizeram pedidos SELECT * FROM clientes WHERE id IN (SELECT DISTINCT cliente_id FROM pedidos); -- Produtos que nunca foram vendidos SELECT * FROM produtos WHERE id NOT IN (SELECT DISTINCT produto_id FROM itens_pedido);

O NOT IN seleciona registros que não correspondem a nenhum dos valores da lista:

-- Clientes que não são das cidades listadas SELECT * FROM clientes WHERE cidade NOT IN ('São Paulo', 'Rio de Janeiro', 'Belo Horizonte');

Atenção!

Se a subconsulta no IN retornar valores NULL, o resultado da consulta será vazio. Use NOT EXISTS ou trate os NULLs explicitamente nesses casos.

IS NULL

25 minutos Intermediário 3 lições

Entendendo valores NULL

NULL é um valor especial em SQL que representa a ausência de dados ou um valor desconhecido. É diferente de zero, string vazia ou qualquer outro valor.

Características importantes do NULL:

  • NULL não é igual a nada, nem mesmo a outro NULL
  • Qualquer operação com NULL resulta em NULL
  • NULL não pode ser comparado com operadores como =, !=, <, >

Ponto-chave

Para verificar se um valor é NULL, você deve usar os operadores IS NULL ou IS NOT NULL. Não use = NULL ou != NULL.

Operador IS NULL

O operador IS NULL é usado para encontrar registros que contêm valores NULL em colunas específicas:

-- Clientes sem telefone cadastrado SELECT * FROM clientes WHERE telefone IS NULL; -- Produtos sem data de validade SELECT * FROM produtos WHERE data_validade IS NULL; -- Pedidos sem data de envio SELECT * FROM pedidos WHERE data_envio IS NULL;

Exemplo de como NULL afeta operações:

-- Esta consulta não retornará registros onde bonus é NULL SELECT * FROM funcionarios WHERE bonus = 0; -- Esta consulta retornará registros onde bonus é NULL SELECT * FROM funcionarios WHERE bonus IS NULL;

Operador IS NOT NULL e tratamento de NULLs

O operador IS NOT NULL é usado para encontrar registros que não contêm valores NULL:

-- Clientes com telefone cadastrado SELECT * FROM clientes WHERE telefone IS NOT NULL; -- Produtos com data de validade definida SELECT * FROM produtos WHERE data_validade IS NOT NULL;

Alguns bancos de dados oferecem funções para tratar valores NULL:

-- Usando COALESCE (retorna o primeiro valor não NULL) SELECT nome, COALESCE(telefone, 'Não informado') AS telefone FROM clientes; -- Usando IFNULL (MySQL) ou ISNULL (SQL Server) SELECT nome, IFNULL(telefone, 'Não informado') AS telefone FROM clientes; -- Usando NVL (Oracle) SELECT nome, NVL(telefone, 'Não informado') AS telefone FROM clientes;

Dica profissional

Evite permitir NULLs em colunas que serão usadas em condições WHERE, JOINs ou cálculos. Sempre que possível, defina valores padrão.

Funções de Agrupamento

45 minutos Avançado 5 lições

Introdução às Funções de Agrupamento

Funções de agregação (ou agrupamento) realizam cálculos em um conjunto de valores e retornam um único valor. Elas são frequentemente usadas com a cláusula GROUP BY.

As principais funções de agregação em SQL são:

  • COUNT() - Conta o número de registros
  • SUM() - Calcula a soma dos valores
  • AVG() - Calcula a média dos valores
  • MIN() - Retorna o valor mínimo
  • MAX() - Retorna o valor máximo

Ponto-chave

Funções de agregação ignoram valores NULL, exceto COUNT(*), que conta todos os registros, incluindo aqueles com NULLs.

Funções COUNT, SUM e AVG

Vamos ver exemplos das funções mais comuns:

-- COUNT: contar registros SELECT COUNT(*) AS total_clientes FROM clientes; SELECT COUNT(email) AS clientes_com_email FROM clientes; -- SUM: somar valores SELECT SUM(valor) AS total_vendas FROM pedidos; SELECT SUM(quantidade * preco) AS valor_estoque FROM itens_pedido; -- AVG: calcular média SELECT AVG(preco) AS preco_medio FROM produtos; SELECT AVG(salario) AS salario_medio FROM funcionarios;

Dica profissional

Use COUNT(1) em vez de COUNT(*) para melhor performance em alguns bancos de dados, pois evita a leitura de todas as colunas.

Funções MIN e MAX

As funções MIN e MAX retornam os valores mínimo e máximo de um conjunto de valores:

-- MIN: valor mínimo SELECT MIN(preco) AS menor_preco FROM produtos; SELECT MIN(data_pedido) AS primeiro_pedido FROM pedidos; -- MAX: valor máximo SELECT MAX(preco) AS maior_preco FROM produtos; SELECT MAX(data_pedido) AS ultimo_pedido FROM pedidos; -- Combinando MIN e MAX SELECT MIN(preco) AS menor_preco, MAX(preco) AS maior_preco, MAX(preco) - MIN(preco) AS diferenca FROM produtos;

Cláusula GROUP BY

A cláusula GROUP BY agrupa registros que têm os mesmos valores em colunas especificadas, permitindo aplicar funções de agregação a cada grupo:

-- Contar clientes por cidade SELECT cidade, COUNT(*) AS total_clientes FROM clientes GROUP BY cidade; -- Valor total de pedidos por cliente SELECT cliente_id, SUM(valor) AS total_compras FROM pedidos GROUP BY cliente_id; -- Preço médio por categoria de produto SELECT categoria, AVG(preco) AS preco_medio FROM produtos GROUP BY categoria;

Atenção!

Todas as colunas no SELECT que não estão em funções de agregação devem estar na cláusula GROUP BY.

Cláusula HAVING e ORDER BY

A cláusula HAVING filtra grupos criados pelo GROUP BY, similar ao que o WHERE faz para registros individuais:

-- Cidades com mais de 10 clientes SELECT cidade, COUNT(*) AS total_clientes FROM clientes GROUP BY cidade HAVING COUNT(*) > 10; -- Categorias com preço médio acima de 100 SELECT categoria, AVG(preco) AS preco_medio FROM produtos GROUP BY categoria HAVING AVG(preco) > 100;

A cláusula ORDER BY ordena o resultado final:

-- Clientes por cidade, ordenados pelo total SELECT cidade, COUNT(*) AS total_clientes FROM clientes GROUP BY cidade ORDER BY total_clientes DESC; -- Produtos por categoria, ordenados pelo preço médio SELECT categoria, AVG(preco) AS preco_medio FROM produtos GROUP BY categoria ORDER BY preco_medio;

Diferença entre WHERE e HAVING

WHERE filtra registros antes do agrupamento; HAVING filtra grupos após o agrupamento.
