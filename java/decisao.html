Estruturas de Decisão em Java

Controle de Fluxo com if, else, switch e Operadores Ternários

1. Estruturas if e else

As estruturas if e else são as estruturas de decisão mais básicas em Java. Elas permitem que um programa execute diferentes blocos de código com base em uma condição booleana (verdadeira ou falsa).

Estrutura if

A estrutura if executa um bloco de código apenas se a condição especificada for verdadeira.

if (condição) {
  // Código a ser executado se a condição for verdadeira
}

Exemplo: Verificar se um número é positivo

int numero = 10;

if (numero > 0) {
  System.out.println("O número é positivo.");
}

Exemplo de Execução:

O número é positivo.

Estrutura else

A estrutura else é usada em conjunto com o if para executar um bloco de código quando a condição do if é falsa.

if (condição) {
  // Código a ser executado se a condição for verdadeira
} else {
  // Código a ser executado se a condição for falsa
}

Exemplo: Verificar se um número é par ou ímpar

int numero = 7;

if (numero % 2 == 0) {
  System.out.println("O número é par.");
} else {
  System.out.println("O número é ímpar.");
}

Exemplo de Execução:

O número é ímpar.

Boas Práticas

  • Indentação: Use sempre indentação para tornar o código mais legível.
  • Chaves: Mesmo para blocos de código com uma única instrução, é recomendado usar chaves {}.
  • Condições claras: Escreva condições que sejam fáceis de entender.

Dica Importante

Evite condições muito complexas dentro de um único if. Se necessário, divida em múltiplas condições ou use variáveis booleanas para armazenar resultados intermediários.

Atividade Prática

Complete o código abaixo para verificar se uma pessoa pode votar (idade >= 16):

  System.out.println("Você pode votar!");
} else {
  System.out.println("Você ainda não pode votar.");
}

2. Estrutura else if

A estrutura else if é usada quando precisamos testar múltiplas condições em sequência. Ela permite verificar uma nova condição apenas se a condição do if anterior for falsa.

Sintaxe do else if

if (condição1) {
  // Código se condição1 for verdadeira
} else if (condição2) {
  // Código se condição1 for falsa e condição2 for verdadeira
} else if (condição3) {
  // Código se condição1 e condição2 forem falsas e condição3 for verdadeira
} else {
  // Código se todas as condições anteriores forem falsas
}

Exemplo: Classificar a nota de um aluno

double nota = 7.5;

if (nota >= 9.0) {
  System.out.println("Conceito A");
} else if (nota >= 7.0) {
  System.out.println("Conceito B");
} else if (nota >= 5.0) {
  System.out.println("Conceito C");
} else {
  System.out.println("Conceito D");
}

Exemplo de Execução:

Conceito B

Exemplo: Verificar a faixa etária

int idade = 25;

if (idade < 12) {
  System.out.println("Criança");
} else if (idade < 18) {
  System.out.println("Adolescente");
} else if (idade < 60) {
  System.out.println("Adulto");
} else {
  System.out.println("Idoso");
}

Exemplo de Execução:

Adulto

Importante: Ordem das Condições

A ordem das condições em uma estrutura else if é muito importante. As condições são verificadas em sequência, e a primeira que for verdadeira será executada, ignorando as demais.

Cuidado com a Ordem!

Considere este exemplo com ordem incorreta:

int idade = 10;

// ORDEM INCORRETA
if (idade > 0) {
  System.out.println("Idade válida");
} else if (idade < 12) {
  // Este bloco nunca será executado!
  System.out.println("Criança");
}

Neste caso, a primeira condição (idade > 0) já é verdadeira para idade = 10, então o programa nunca verificará a segunda condição.

Dica

Quando usar múltiplos else if, organize as condições da mais específica para a mais geral, ou da mais restritiva para a menos restritiva.

Atividade Prática

Organize as condições abaixo na ordem correta para classificar a temperatura:

Temperatura > 30°C: Muito quente
Temperatura < 10°C: Muito frio
Temperatura entre 10°C e 20°C: Frio
Temperatura entre 20°C e 30°C: Agradável

3. Operadores Lógicos

Os operadores lógicos são usados para combinar múltiplas condições em estruturas de decisão. Eles permitem criar expressões booleanas mais complexas, avaliando duas ou mais condições simultaneamente.

Principais Operadores Lógicos

&&
E (AND)
Retorna true apenas se todas as condições forem verdadeiras
||
OU (OR)
Retorna true se pelo menos uma condição for verdadeira
!
NÃO (NOT)
Inverte o valor lógico da condição

Operador E (AND) - &&

O operador && (E lógico) retorna true apenas se todas as condições envolvidas forem verdadeiras. Se pelo menos uma condição for falsa, o resultado será false.

Tabela Verdade do AND (&&)

Condição A Condição B A && B
true true true
true false false
false true false
false false false

Exemplo: Verificar se um número está em um intervalo

int numero = 15;

// Verifica se o número está entre 10 e 20 (inclusive)
if (numero >= 10 && numero <= 20) {
  System.out.println("O número está entre 10 e 20.");
} else {
  System.out.println("O número não está entre 10 e 20.");
}

Exemplo de Execução:

O número está entre 10 e 20.

Exemplo: Verificar condições para empréstimo

boolean temRendaComprovada = true;
boolean temNomeLimpo = true;
boolean temGarantia = false;

// Para aprovar o empréstimo, precisa ter renda comprovada E nome limpo
if (temRendaComprovada && temNomeLimpo) {
  System.out.println("Empréstimo aprovado!");
} else {
  System.out.println("Empréstimo negado.");
}

Exemplo de Execução:

Empréstimo aprovado!

Operador OU (OR) - ||

O operador || (OU lógico) retorna true se pelo menos uma das condições for verdadeira. Retorna false apenas se todas as condições forem falsas.

Tabela Verdade do OR (||)

Condição A Condição B A || B
true true true
true false true
false true true
false false false

Exemplo: Verificar se um caractere é vogal

char letra = 'a';

// Verifica se a letra é uma vogal (minúscula ou maiúscula)
if (letra == 'a' || letra == 'e' || letra == 'i' ||
    letra == 'o' || letra == 'u' || letra == 'A' ||
    letra == 'E' || letra == 'I' || letra == 'O' ||
    letra == 'U') {
  System.out.println("A letra é uma vogal.");
} else {
  System.out.println("A letra é uma consoante.");
}

Exemplo de Execução:

A letra é uma vogal.

Exemplo: Verificar condições para desconto

boolean ehIdoso = false;
boolean ehEstudante = true;
boolean ehDeficiente = false;

// Concede desconto se for idoso OU estudante OU deficiente
if (ehIdoso || ehEstudante || ehDeficiente) {
  System.out.println("Desconto concedido!");
} else {
  System.out.println("Sem direito a desconto.");
}

Exemplo de Execução:

Desconto concedido!

Operador NÃO (NOT) - !

O operador ! (NÃO lógico) inverte o valor lógico de uma condição. Se a condição for true, ele retorna false, e vice-versa.

Tabela Verdade do NOT (!)

Condição A !A
true false
false true

Exemplo: Verificar se um número não é positivo

int numero = -5;

// Verifica se o número NÃO é positivo
if (!(numero > 0)) {
  System.out.println("O número não é positivo.");
} else {
  System.out.println("O número é positivo.");
}

Exemplo de Execução:

O número não é positivo.

Exemplo: Verificar se um campo não está vazio

String nome = "";

// Verifica se o nome NÃO está vazio
if (!nome.isEmpty()) {
  System.out.println("Nome: " + nome);
} else {
  System.out.println("O nome não pode ser vazio!");
}

Exemplo de Execução:

O nome não pode ser vazio!

Combinando Operadores Lógicos

Podemos combinar múltiplos operadores lógicos para criar condições mais complexas. Nesses casos, é importante entender a precedência dos operadores:

  1. ! (NÃO) - Maior precedência
  2. && (E)
  3. || (OU) - Menor precedência

Exemplo: Sistema de login

String usuario = "admin";
String senha = "12345";
boolean contaAtiva = true;
boolean contaBloqueada = false;

// Login válido se: usuário E senha corretos E conta ativa E NÃO bloqueada
if (usuario.equals("admin") && senha.equals("12345") &&
    contaAtiva && !contaBloqueada) {
  System.out.println("Login realizado com sucesso!");
} else {
  System.out.println("Usuário ou senha incorretos, ou conta inativa/bloqueada.");
}

Exemplo de Execução:

Login realizado com sucesso!

Exemplo: Verificar elegibilidade para vaga

int idade = 22;
boolean temEnsinoSuperior = true;
boolean temExperiencia = false;
boolean ehEstagiario = true;

// Elegível se: (idade entre 18 e 30) E (tem ensino superior OU é estagiário)
if (idade >= 18 && idade <= 30 &&
    (temEnsinoSuperior || ehEstagiario)) {
  System.out.println("Candidato elegível para a vaga.");
} else {
  System.out.println("Candidato não elegível para a vaga.");
}

Exemplo de Execução:

Candidato elegível para a vaga.

Lei de De Morgan

A Lei de De Morgan é um conjunto de regras que relacionam os operadores E e OU através da negação. Ela é útil para simplificar expressões lógicas complexas:

// Primeira lei: !(A && B) = !A || !B
// Segunda lei: !(A || B) = !A && !B

Exemplo: Aplicando a Lei de De Morgan

boolean temDinheiro = false;
boolean temCartao = true;

// Expressão original
if (!(temDinheiro && temCartao)) {
  System.out.println("Não pode comprar");
}

// Equivalente usando a Lei de De Morgan
if (!temDinheiro || !temCartao) {
  System.out.println("Não pode comprar");
}

Dica de Legibilidade

Para condições muito complexas, considere usar variáveis booleanas intermediárias para tornar o código mais legível:

boolean idadeValida = idade >= 18 && idade <= 30;
boolean qualificacaoValida = temEnsinoSuperior || ehEstagiario;

if (idadeValida && qualificacaoValida) {
  System.out.println("Candidato elegível para a vaga.");
}

Cuidado com Avaliação de Curto-Circuito!

Os operadores && e || usam avaliação de curto-cuito:

  • &&: Se a primeira condição for falsa, a segunda não é avaliada
  • ||: Se a primeira condição for verdadeira, a segunda não é avaliada

Isso pode causar problemas se a segunda condição tiver efeitos colaterais:

int x = 0;
int y = 5;

// Cuidado: divisão por zero não ocorre devido ao curto-circuito
if (x != 0 && (y / x) > 2) {
  System.out.println("Condição atendida");
}

4. Estrutura switch case

A estrutura switch case é uma alternativa ao else if quando precisamos comparar uma única variável com múltiplos valores constantes. Ela torna o código mais limpo e legível em certas situações.

Sintaxe do switch case

switch (variável) {
  case valor1:
    // Código para valor1
    break;
  case valor2:
    // Código para valor2
    break;
  case valor3:
    // Código para valor3
    break;
  default:
    // Código padrão (nenhum dos casos acima)
    break;
}

Exemplo: Dia da semana

int dia = 3;
String nomeDia;

switch (dia) {
  case 1:
    nomeDia = "Domingo";
    break;
  case 2:
    nomeDia = "Segunda-feira";
    break;
  case 3:
    nomeDia = "Terça-feira";
    break;
  case 4:
    nomeDia = "Quarta-feira";
    break;
  case 5:
    nomeDia = "Quinta-feira";
    break;
  case 6:
    nomeDia = "Sexta-feira";
    break;
  case 7:
    nomeDia = "Sábado";
    break;
  default:
    nomeDia = "Dia inválido";
    break;
}

System.out.println("Hoje é " + nomeDia);

Exemplo de Execução:

Hoje é Terça-feira

A Importância do break

O comando break é essencial em cada case. Sem ele, o programa continuaria executando os próximos casos mesmo que a condição não seja verdadeira. Isso é chamado de fallthrough.

Cuidado!

Se você esquecer o break, o código continuará executando os próximos casos:

int opcao = 1;

switch (opcao) {
  case 1:
    System.out.println("Opção 1 selecionada");
    // Sem break!
  case 2:
    System.out.println("Opção 2 selecionada");
    break;
}

Saída (incorreta):

Opção 1 selecionada
Opção 2 selecionada

Switch com Strings (Java 7+)

A partir do Java 7, é possível usar switch com variáveis do tipo String:

String mes = "Janeiro";
int dias;

switch (mes.toLowerCase()) {
  case "janeiro":
  case "março":
  case "maio":
  case "julho":
  case "agosto":
  case "outubro":
  case "dezembro":
    dias = 31;
    break;
  case "abril":
  case "junho":
  case "setembro":
  case "novembro":
    dias = 30;
    break;
  case "fevereiro":
    dias = 28; // Simplificado
    break;
  default:
    dias = 0;
    break;
}

System.out.println("O mês de " + mes + " tem " + dias + " dias.");

Exemplo de Execução:

O mês de Janeiro tem 31 dias.

Switch vs if-else

Característica switch case if-else
Tipo de comparação Apenas igualdade com valores constantes Qualquer condição booleana
Variáveis suportadas int, byte, short, char, String (Java 7+), enum Qualquer tipo
Legibilidade Mais limpo para múltiplos valores Pode ficar verboso com muitas condições
Flexibilidade Menos flexível Muito flexível
Performance Geralmente mais rápido para muitos casos Pode ser mais lento com muitas condições

Quando usar switch case?

Use switch case quando:

  • Você está comparando uma única variável com múltiplos valores constantes
  • Os valores são conhecidos em tempo de compilação
  • Você quer um código mais limpo e legível

Atividade Prática

Complete o código abaixo para criar um menu de opções usando switch case:

int opcao = 2;

switch (opcao) {
  case 1:
    System.out.println("Cadastro de clientes");
    break;
  case :
    System.out.println();
    break;
  case 3:
    System.out.println("Relatórios");
    break;
  default:
    System.out.println();
    break;
}

5. Operador Ternário

O operador ternário é uma forma compacta de escrever uma estrutura if-else simples. Ele é chamado de "ternário" porque usa três operandos.

Sintaxe do Operador Ternário

variavel = (condição) ? valor_se_verdadeiro : valor_se_falso;

Como Funciona

  • condição: Uma expressão booleana que retorna verdadeiro ou falso
  • valor_se_verdadeiro: Valor retornado se a condição for verdadeira
  • valor_se_falso: Valor retornado se a condição for falsa

Exemplo: Verificar se um número é positivo

// Usando if-else tradicional
int numero = 10;
String mensagem1;

if (numero > 0) {
  mensagem1 = "Positivo";
} else {
  mensagem1 = "Negativo ou zero";
}

// Usando operador ternário
String mensagem2 = (numero > 0) ? "Positivo" : "Negativo ou zero";

Exemplo: Determinar o maior número

int a = 15;
int b = 20;

// Usando if-else tradicional
int maior1;
if (a > b) {
  maior1 = a;
} else {
  maior1 = b;
}

// Usando operador ternário
int maior2 = (a > b) ? a : b;

System.out.println("O maior número é: " + maior2);

Exemplo de Execução:

O maior número é: 20

Operador Ternário Aninhado

É possível aninhar operadores ternários para criar estruturas mais complexas, semelhantes ao else if:

int nota = 7;
String conceito = (nota >= 9) ? "A" :
               (nota >= 7) ? "B" :
               (nota >= 5) ? "C" : "D";

System.out.println("Conceito: " + conceito);

Exemplo de Execução:

Conceito: B

Vantagens e Desvantagens

Aspecto Vantagens Desvantagens
Concisão Código mais curto e direto Pode ficar difícil de ler com lógica complexa
Legibilidade Boa para condições simples Aninhamentos excessivos prejudicam a legibilidade
Uso Perfeito para atribuições condicionais simples Não substitui estruturas if-else complexas
Performance Geralmente equivalente ao if-else Não há ganho de performance significativo

Boas Práticas

  • Use o operador ternário apenas para condições simples e diretas
  • Avoid aninhamentos muito profundos (mais de 2 níveis)
  • Se a lógica ficar complexa, prefira o if-else tradicional
  • O operador ternário é ideal para atribuições condicionais em uma única linha

Cuidado com a Legibilidade!

Operadores ternários muito complexos podem tornar o código difícil de entender:

// Exemplo de uso inadequado
String resultado = (x > 0) ? ((y > 0) ? "Ambos positivos" : "X positivo, Y negativo") :
                      ((y > 0) ? "X negativo, Y positivo" : "Ambos negativos");

Neste caso, um if-else tradicional seria muito mais claro!

Dica

O operador ternário é especialmente útil em expressões de retorno de métodos e em inicializações de variáveis que dependem de uma condição simples.
