1. Modularidade

Modularidade é um princípio fundamental da programação que consiste em dividir um programa complexo em partes menores e mais gerenciáveis, chamadas de módulos. Em Java, esses módulos são implementados principalmente através de métodos e classes.

Divisão do Programa em Partes Menores

A modularidade permite que um programa seja dividido em unidades lógicas que realizam tarefas específicas. Cada módulo deve ter uma única responsabilidade bem definida, o que facilita o desenvolvimento, teste e manutenção do código.

Uso de Métodos public static como Módulos

Em Java, especialmente em programas inicias, utilizamos métodos estáticos (public static) como módulos para organizar o código. Cada método pode ser visto como um módulo independente que realiza uma tarefa específica.

public class ProgramaModular {
  public static void main(String[] args) {
    // Módulo principal que coordena outros módulos
    double[] notas = obterNotas(); // Módulo de entrada
    double media = calcularMedia(notas); // Módulo de processamento
    exibirResultado(media); // Módulo de saída
  }

  // Módulo para obter notas
  public static double[] obterNotas() {
    // Implementação...
  }

  // Módulo para calcular média
  public static double calcularMedia(double[] notas) {
    // Implementação...
  }

  // Módulo para exibir resultado
  public static void exibirResultado(double media) {
    // Implementação...
  }
}

Princípios da Boa Modularização

  • Alta coesão: Cada módulo deve ter uma única responsabilidade bem definida.
  • Baixo acoplamento: Módulos devem ser o mais independentes possível uns dos outros.
  • Interface clara: Cada módulo deve ter uma interface bem definida para interação com outros módulos.
  • Tamanho adequado: Módulos não devem ser nem muito grandes (difíceis de entender) nem muito pequenos (excesso de fragmentação).

Exemplo Prático de Modularização

Vamos considerar um programa que calcula estatísticas básicas de um conjunto de números:

Programa Não Modular

public class Estatisticas {
  public static void main(String[] args) {
    double[] numeros = {5.5, 8.2, 3.7, 9.1, 6.3};
    double soma = 0;
    for (double num : numeros) {
      soma += num;
    }
    double media = soma / numeros.length;
    double maior = numeros[0];
    for (double num : numeros) {
      if (num > maior) {
        maior = num;
      }
    }
    System.out.println("Média: " + media);
    System.out.println("Maior: " + maior);
  }
}

Dica de Modularização

Uma boa prática é identificar as operações principais do seu programa e transformar cada uma delas em um método separado. O método main deve servir apenas como um coordenador, chamando os outros métodos na ordem adequada.

Atividade Prática

Identifique quais partes do código abaixo poderiam ser transformadas em módulos separados:

public class ConversorTemperatura {
  public static void main(String[] args) {
    Scanner scanner = new Scanner(System.in);
    System.out.print("Digite a temperatura em Celsius: ");
    double celsius = scanner.nextDouble();
    double fahrenheit = celsius * 1.8 + 32;
    System.out.println("Temperatura em Fahrenheit: " + fahrenheit);
    scanner.close();
  }
}

2. Funções e Procedimentos (Métodos public static)

Em Java, funções e procedimentos são implementados como métodos. A principal diferença entre eles é que funções retornam um valor, enquanto procedimentos não (retornam void). Vamos explorar a estrutura e o uso desses métodos.

Estrutura Básica de um Método

Um método em Java tem a seguinte estrutura básica:

// Modificador de acesso + Modificador estático + Tipo de retorno + Nome + Parâmetros
public static tipoRetorno nomeMetodo(tipoParametro1 param1, tipoParametro2 param2) {
  // Corpo do método
  // Declarações e instruções
  return valor; // Opcional, dependendo do tipo de retorno
}

Procedimentos (void)

Procedimentos são métodos que não retornam nenhum valor. Eles são identificados pelo uso da palavra-chave void como tipo de retorno. Geralmente são usados para executar ações, como exibir informações na tela ou modificar o estado de variáveis.

// Exemplo de procedimento
public static void exibirMensagem(String mensagem) {
  System.out.println("**********");
  System.out.println(mensagem);
  System.out.println("**********");
}

// Chamada do procedimento
exibirMensagem("Olá, mundo!");

Funções (retornam valor via return)

Funções são métodos que retornam um valor após sua execução. O tipo de retorno deve ser especificado na declaração do método, e o valor deve ser retornado usando a palavra-chave return.

// Exemplo de função
public static int somar(int a, int b) {
  int resultado = a + b;
  return resultado;
}

// Chamada da função
int soma = somar(5, 3); // soma recebe 8
System.out.println("Resultado: " + soma);

Chamadas de Métodos

Para utilizar um método, precisamos fazer uma chamada a ele. A forma de chamada depende se o método retorna um valor ou não:

Chamada de Procedimento

// Procedimento que não retorna valor
public static void exibirTabuada(int numero) {
  for (int i = 1; i <= 10; i++) {
    System.out.println(numero + " x " + i + " = " + (numero * i));
  }
}

// Chamada do procedimento
exibirTabuada(7);

Chamada de Função

// Função que retorna um valor
public static double calcularMedia(double[] valores) {
  double soma = 0;
  for (double valor : valores) {
    soma += valor;
  }
  return soma / valores.length;
}

// Chamada da função
double[] notas = {8.5, 7.2, 9.0};
double media = calcularMedia(notas);

Uma boa prática é manter o método main o mais simples possível, usando-o apenas para coordenar as chamadas a outros métodos. Isso torna o código mais organizado e fácil de entender.

Assinatura de Métodos

A assinatura de um método é composta pelo seu nome e pela lista de parâmetros (tipos e ordem). Java permite que múltiplos métodos tenham o mesmo nome, desde que tenham assinaturas diferentes. Isso é chamado de sobrecarga de métodos.

// Sobrecarga de métodos
public static int somar(int a, int b) {
  return a + b;
}

public static double somar(double a, double b) {
  return a + b;
}

public static int somar(int a, int b, int c) {
  return a + b + c;
}

// Chamadas
int resultado1 = somar(5, 3); // Chama o primeiro método
double resultado2 = somar(2.5, 3.7); // Chama o segundo método
int resultado3 = somar(4, 2, 6); // Chama o terceiro método

Dica sobre Métodos

Cada método deve ter uma única responsabilidade. Se um método está fazendo muitas coisas, considere dividi-lo em métodos menores e mais específicos.

3. Escopo de Variáveis

O escopo de uma variável determina onde ela pode ser acessada e modificada no código. Em Java, existem diferentes tipos de escopo, cada um com suas próprias regras e características.

Escopo Local

Variáveis declaradas dentro de um método têm escopo local, o que significa que só podem ser acessadas dentro desse método. Elas são criadas quando o método é chamado e destruídas quando o método termina sua execução.

public class ExemploEscopoLocal {
  public static void metodoExemplo() {
    int variavelLocal = 10; // Escopo local
    System.out.println("Dentro do método: " + variavelLocal);
  }

  public static void main(String[] args) {
    metodoExemplo();
    // System.out.println(variavelLocal); // Erro! variavelLocal não é acessível aqui
  }
}

Escopo de Bloco { }

Variáveis declaradas dentro de um bloco de código (delimitado por chaves { }) têm escopo limitado a esse bloco. Isso inclui estruturas como if, for, while, etc.

public static void exemploBloco() {
  int x = 10; // Disponível em todo o método

  if (x > 5) {
    int y = 20; // Disponível apenas dentro deste bloco if
    System.out.println("Dentro do if: " + y);
  }

  // System.out.println(y); // Erro! y não é acessível fora do bloco if

  for (int i = 0; i < 3; i++) {
    int z = i * 2; // Disponível apenas dentro deste bloco for
    System.out.println("Dentro do for: " + z);
  }

  // System.out.println(z); // Erro! z não é acessível fora do bloco for
}

Escopo de Parâmetros

Os parâmetros de um método têm escopo local ao método, podendo ser acessados em qualquer parte dentro dele. Eles funcionam como variáveis locais que recebem os valores passados na chamada do método.

public static void exemploParametros(int a, String b) {
  // a e b são parâmetros com escopo local ao método
  System.out.println("Parâmetro a: " + a);
  System.out.println("Parâmetro b: " + b);
}

public static void main(String[] args) {
  exemploParametros(10, "Olá");
  // System.out.println(a); // Erro! a não é acessível aqui
}

Escopo Global static (evitar)

Variáveis declaradas fora de qualquer método, mas dentro da classe e com o modificador static, têm escopo global e podem ser acessadas por qualquer método da classe. No entanto, o uso excessivo de variáveis globais é considerado uma má prática de programação.

public class ExemploGlobal {
  private static int contador = 0; // Variável global estática

  public static void incrementar() {
    contador++; // Acessando a variável global
  }

  public static void exibir() {
    System.out.println("Valor do contador: " + contador);
  }

  public static void main(String[] args) {
    incrementar();
    incrementar();
    exibir(); // Exibe: Valor do contador: 2
  }
}

Cuidado com Variáveis Globais!

O uso excessivo de variáveis globais pode tornar o código difícil de entender, manter e depurar. Prefira passar dados como parâmetros sempre que possível.

Boas Práticas de Escopo

  • Declare variáveis no menor escopo possível: Isso reduz a chance de erros e torna o código mais claro.
  • Evite variáveis globais: Use-as apenas quando estritamente necessário e com moderação.
  • Nomes significativos: Use nomes claros e descritivos para variáveis, especialmente quando houver variáveis com o mesmo nome em escopos diferentes.
  • Inicialize variáveis: Sempre inicialize variáveis locais antes de usá-las.

Sombreamento de Variáveis (Variable Shadowing)

Ocorre quando uma variável em um escopo interno tem o mesmo nome de uma variável em um escopo externo. Nesse caso, a variável do escopo interno "sombra" a do escopo externo.

public static void exemploSombreamento() {
  int x = 10; // Variável do escopo do método
  System.out.println("x externo: " + x); // Exibe 10

  if (true) {
    int x = 20; // Variável do escopo do bloco (sombra a externa)
    System.out.println("x interno: " + x); // Exibe 20
  }

  System.out.println("x externo novamente: " + x); // Exibe 10
}

Dica sobre Escopo

O sombreamento de variáveis pode tornar o código confuso. Evite usar o mesmo nome para variáveis em escopos diferentes, a menos que tenha uma boa razão para isso.

Exemplo Prático de Escopos

public class ExemploEscopos {
  private static int global = 100; // Escopo global

  public static void main(String[] args) {
    int localMain = 10; // Escopo local ao main
    System.out.println("global: " + global);
    System.out.println("localMain: " + localMain);

    metodoExemplo(localMain);
  }

  public static void metodoExemplo(int parametro) {
    // parametro tem escopo local ao método
    int localMetodo = 20; // Escopo local ao método
    System.out.println("global: " + global);
    System.out.println("parametro: " + parametro);
    System.out.println("localMetodo: " + localMetodo);

    if (parametro > 5) {
      int localBloco = 30; // Escopo do bloco if
      System.out.println("localBloco: " + localBloco);
    }

    // System.out.println(localBloco); // Erro! localBloco não é acessível aqui
  }
}

4. Passagem de Parâmetros

A passagem de parâmetros é o mecanismo pelo qual dados são transferidos entre métodos. Em Java, existem duas formas principais de passagem de parâmetros: por valor (para tipos primitivos) e por referência (para objetos, incluindo arrays).

Passagem por Valor (Primitivos)

Quando passamos tipos primitivos (int, double, char, boolean, etc.) como parâmetros, estamos fazendo uma passagem por valor. Isso significa que uma cópia do valor é criada e passada para o método. Qualquer modificação nessa cópia não afeta a variável original.

public class PassagemPorValor {
  public static void main(String[] args) {
    int numero = 10;
    System.out.println("Antes do método: " + numero); // 10

    tentarModificar(numero);

    System.out.println("Depois do método: " + numero); // 10 (não mudou)
  }

  public static void tentarModificar(int valor) {
    System.out.println("Dentro do método (antes): " + valor); // 10
    valor = 20; // Modifica apenas a cópia local
    System.out.println("Dentro do método (depois): " + valor); // 20
  }
}

Cópia de Referência para Arrays

Quando passamos arrays (ou qualquer objeto) como parâmetros, estamos passando uma cópia da referência ao objeto original. Isso significa que o método pode modificar o conteúdo do array, e essas modificações serão visíveis fora do método.

public class PassagemArray {
  public static void main(String[] args) {
    int[] numeros = {1, 2, 3};
    System.out.println("Antes do método: " + Arrays.toString(numeros)); // [1, 2, 3]

    modificarArray(numeros);

    System.out.println("Depois do método: " + Arrays.toString(numeros)); // [10, 2, 3]
  }

  public static void modificarArray(int[] array) {
    System.out.println("Dentro do método (antes): " + Arrays.toString(array)); // [1, 2, 3]
    array[0] = 10; // Modifica o array original
    System.out.println("Dentro do método (depois): " + Arrays.toString(array)); // [10, 2, 3]
  }
}

Parâmetros Formais e Reais

Na passagem de parâmetros, distinguimos entre:

  • Parâmetros formais: São as variáveis declaradas na assinatura do método.
  • Parâmetros reais (ou argumentos): São os valores ou variáveis passados na chamada do método.
public static void exemplo(int a, int b) { // a e b são parâmetros formais   System.out.println("a = " + a + ", b = " + b);
}

public static void main(String[] args) {
  int x = 5;
  int y = 10;
  exemplo(x, y); // x e y são parâmetros reais
  exemplo(20, 30); // 20 e 30 são parâmetros reais
}

Passagem de Múltiplos Parâmetros

Java permite a passagem de múltiplos parâmetros para um método. A ordem dos parâmetros na chamada deve corresponder à ordem na declaração do método.

public static String criarSaudacao(String nome, int idade, boolean formal) {
  String saudacao = formal ? "Prezado(a) " : "Olá, ";
  return saudacao + nome + "! Você tem " + idade + " anos.";
}

public static void main(String[] args) {
  String mensagem1 = criarSaudacao("Ana", 25, true);
  String mensagem2 = criarSaudacao("Carlos", 30, false);
  System.out.println(mensagem1);
  System.out.println(mensagem2);
}

Parâmetros com Valores Padrão (Simulação)

Java não suporta diretamente parâmetros com valores padrão como algumas outras linguagens. No entanto, podemos simular esse comportamento usando sobrecarga de métodos.

public static void exibirMensagem(String mensagem) {
  exibirMensagem(mensagem, false); // Chama a versão com dois parâmetros }

public static void exibirMensagem(String mensagem, boolean uppercase) {
  if (uppercase) {
    System.out.println(mensagem.toUpperCase());
  } else {
    System.out.println(mensagem);
  }
}

public static void main(String[] args) {
  exibirMensagem("Olá, mundo!"); // Usa o valor padrão (false)
  exibirMensagem("Olá, mundo!", true); // Especifica o valor explicitamente }

Passagem de Arrays como Parâmetros

Arrays são frequentemente passados como parâmetros para métodos que realizam operações em conjunto de dados, como cálculos estatísticos, ordenação, busca, etc.

public class OperacoesArray {
  public static void main(String[] args) {
    double[] notas = {7.5, 8.2, 6.9, 9.1, 7.8};

    double media = calcularMedia(notas);
    double maior = encontrarMaior(notas);
    double menor = encontrarMenor(notas);

    System.out.println("Média: " + media);
    System.out.println("Maior nota: " + maior);
    System.out.println("Menor nota: " + menor);
  }

  public static double calcularMedia(double[] valores) {
    double soma = 0;
    for (double valor : valores) {
      soma += valor;
    }
    return soma / valores.length;
  }

  public static double encontrarMaior(double[] valores) {
    double maior = valores[0];
    for (double valor : valores) {
      if (valor > maior) {
        maior = valor;
      }
    }
    return maior;
  }

  public static double encontrarMenor(double[] valores) {
    double menor = valores[0];
    for (double valor : valores) {
      if (valor < menor) {
        menor = valor;
      }
    }
    return menor;
  }
}

Dica sobre Passagem de Parâmetros

Lembre-se: para tipos primitivos, o método recebe uma cópia do valor. Para objetos (incluindo arrays), o método recebe uma cópia da referência, mas pode modificar o conteúdo do objeto original.
